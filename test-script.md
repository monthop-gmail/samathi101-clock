# Samathi101 Clock Test Script (QA Checklist)

เอกสารนี้ใช้สำหรับตรวจสอบการทำงาน (Test Script) ของแอปจับเวลาสมาธิ **ทุกครั้งก่อน Deploy ขึ้น Production** เพื่อป้องกันบั๊กลุกลาม

## 1. การจับเวลามาตรฐาน (Timer Core Flow)
- [ ] **เปิดแอป**: โหลดได้ปกติ ไม่มี Error ใน Console
- [ ] **กดเริ่ม (เริ่มต้นใหม่)**: 
  - [ ] ต้องได้ยิน **"เสียงระฆัง"** ทันทีตั้งแต่คลิกแรก
  - [ ] ตัวเลขนาที/เลขอิเล็กทรอนิกส์ นับถอยหลังถูกต้องทีละ 1 วินาที
  - [ ] Animation วงกลมลดลงแบบ Smooth
- [ ] **กดหยุดชั่วคราว (Pause)**:
  - [ ] เวลาหยุดนิ่ง 
  - [ ] หน้าจอยังสว่างอยู่ (Wake Lock สานต่อการทำงานได้ปกติ ไม่ปล่อยให้จอดับ)
- [ ] **กดดำเนินการต่อ (Resume)**:
  - [ ] เวลานับต่อจากจุดเดิม 
  - [ ] **ต้องไม่มีเสียงระฆัง** แทรกขึ้นมาตอนกดต่อ (เพราะถือว่ายังเป็น Session เดิม)
- [ ] **กดรีเซ็ต (Reset)**:
  - [ ] เวลากลับไปค่าตั้งต้นของ Preset ล่าสุดที่เลือกไว้ (เช่น 15 นาที)

## 2. โหมดเตือนหลับ (Sleep Alert)
> **Logic พื้นฐาน:** ต้องดังทุกๆ 5 นาที (เมื่อเวลาที่เหลือหาร 5 นาทีลงตัว) ยกเว้นตอนเริ่มต้นและตอนจบ (00:00) 
- [ ] **เตือนหลับแบบยาว 15 นาที**:
  - เลือก Preset จับเวลา (15 นาที)
  - กด "เริ่ม"
  - ใช้ `Date.now` hack หรือจับเวลาจริง -> เมื่อเวลาเหลือ 10:00 ต้องได้ยินเสียงเด้ง (Bounce)
  - เมื่อเวลาเหลือ 05:00 ต้องได้ยินเสียงเด้ง (Bounce)
- [ ] **การปิดเตือนหลับทำงานจริง**:
  - กดปุ่ม "ตื่นตัว: ปิด" (ให้สถานะแปลว่า ไม่เปิดเตือนหลับ)
  - เมื่อเวลาเหลือ 10:00 (หรือหาร 5 ลงตัว) **ต้องไม่ได้ยิน** เสียงเตือน

## 3. เสียงนำนั่งสมาธิและเดินจงกรม (Guided Audios)
- [ ] **ทดสอบโหมดเดินจงกรม**:
  - กดเลือกหมวด **"เดินจงกรม 30 นาที"**
  - กดปุ่ม **"เปิดเสียงนำเดินจงกรม"**
  - กดเริ่ม -> **ต้องได้ยินเสียงคำสอนเดินจงกรม** แทรกพร้อมกับระฆังเริ่มต้น
- [ ] **ทดสอบจบโหมดเดินจงกรม**:
  - ให้เวลาถึง 00:00 น.
  - **ต้องได้ยินเสียงระฆังจบชุด** จากนั้นตามด้วย **เสียงคำสอนแผ่เมตตาท้ายเดินจงกรม**
- [ ] **ทดสอบเหมือนกันกับหมวด "นั่งสมาธิ"**
  - ต้องได้ยินเสียงนำนั่งสมาธิอย่างถูกต้อง

## 4. ปัญหาหน้าจอดับ (Screen Timeout / Wake Lock)
- [ ] ทดสอบในมือถือจริง (iOS / Android)
- [ ] ในขณะที่จับเวลา (กดปุ่มเริ่มแล้ว) ปล่อยหน้าจอมือถือทิ้งไว้ 5-10 นาทีโดยไม่ต้องสัมผัส
- [ ] **ความถูกต้อง:** หน้าจอต้อง **ไม่ดับ** และสว่างค้างไว้ตลอดการจับเวลา
- [ ] ขณะจับเวลา ให้กด pause -> หน้าจอต้องยังไม่ดับ

## 5. การแจ้งเตือนอัปเดตแอป PWA (SW Update Overlay)
- [ ] เมื่อโค้ดมีการเปลี่ยน `CACHE_NAME` ใหม่ใน `sw.js` 
- [ ] เปิดหน้าแอปทิ้งไว้ / หรือรีเฟรชหน้า
- [ ] ต้องมีแถบ Pop-up เลื่อนขึ้นมาตรงกลางล่างบอกว่า **"🔄 มีเวอร์ชันใหม่พร้อมใช้งาน"**
- [ ] เมื่อกด **"รีเฟรชเลย"** แอปต้องรีโหลดและได้เวอร์ชันใหม่ (สังเกตเลขเวอร์ชันมุมซ้าย/ขวาบน)

## 6. Offline Mode
- [ ] โหลดแอปเสร็จ ปิดอินเทอร์เน็ต (Airplane Mode)
- [ ] เครื่องยังต้องทำงานได้ 100% (รวมถึงเสียงต่างๆ เพราะแคชไว้หมดแล้ว)
